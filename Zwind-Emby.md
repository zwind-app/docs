# 用 Zwind 播放 Emby 媒体库资源

Zwind 是一个功能强大的 WebDAV Server 软件，目前支持 iOS/iPadOS。
通过 Zwind 的解析器插件能力，可以将多种类型的媒体内容挂载进 WebDAV 目录树中。

对 Emby 用户来说，它主要可以做两件事：
1. 在 iPhone 或 iPad 上直接浏览、搜索和播放 Emby 里的电影、剧集；
2. 把 Emby 转成 WebDAV 入口，让支持 WebDAV 的电视、播放器或其他设备访问媒体库。

你可以把它理解成：

**Emby 服务器 → Zwind WebDAV 目录 → Zwind 播放器或第三方 WebDAV 客户端**

## 使用前准备

1. 安装 [Zwind](https://apps.apple.com/app/id6755239096)。
2. Zwind 内购“Emby 媒体库解析器”已解锁（访问 [Telegram：@zwindapp](https://t.me/zwindapp/) 获取限时兑换码）。

## 第一步：在 Zwind 中创建本地 WebDAV 服务器

1. 打开 Zwind。
2. 点击左下角的 **“加号”** 按钮。
3. 选择 **“创建新服务器”**。
4. 存储类型选择 **“本地文件系统”**。
5. 点击 **“保存”**。

回到首页后：

1. 选中刚刚创建的服务器。
2. 再次点击左下角的 **“加号”** 按钮。
3. 选择 **“给当前服务器添加文件夹”**。
4. 选择一个本地文件夹或 iCloud 文件夹。

## 第二步：启动服务器，并创建 Emby 文件夹

1. 回到 Zwind 首页。
2. 选中刚刚创建的 WebDAV 服务器。
3. 点击右下角的启动按钮。
4. 服务器启动后，Zwind 会打开自带的 WebDAV Browser。
5. 进入你刚才添加的数据文件夹。
6. 点击右下角的 **“加号”** 按钮。
7. 选择 **“新建文件夹”**。
8. 文件夹名称输入 **Emby**。
9. 点击 **“创建”**。

## 第三步：给 Emby 文件夹绑定解析器

1. 在 Browser 中长按刚刚创建的 **Emby** 文件夹。
2. 选择 **“配置解析器绑定”**。
3. 在 **“解析器”** 中选择 **“Emby 媒体库解析器”**。

然后填写 Emby 连接信息：

1. **Emby Endpoints**：填写 Emby Server 地址。
2. **用户名**：填写 Emby 用户名。
3. **密码**：填写 Emby 密码。

填写完成后点击 **“保存”**。

## 第四步：浏览 Emby 媒体库

重新进入 **Emby** 文件夹后，你会看到 Zwind 自动生成的 Emby 目录，例如：

| 目录 | 用途 |
| --- | --- |
| **Libraries** | 按 Emby 媒体库浏览，例如电影库、剧集库。 |
| **Recently Added** | 最近入库的媒体。 |
| **Continue Watching** | 当前 Emby 用户可以继续观看的内容。 |
| **Movies**、**Series** | 按电影、剧集聚合浏览。 |
| **Genres**、**People**、**Studios**、**Collections** | 按类型、人物、制作方、合集浏览 |
| **Search** | 用文件夹名称发起 Emby 搜索。 |

进入某个媒体条目后，通常会看到：

1. 可播放的视频文件；
2. `.strm` 文件；
3. `metadata.json` 和 `metadata.md`；
4. `poster.jpg` 图片；
5. `playback.json`。

你只需要点击可播放的视频文件，就可以在 Zwind 中播放。

## 使用 Search 搜索 Emby 内容

Zwind 的 Emby 搜索不是输入框，而是一个文件夹操作。

1. 进入 **Emby** 文件夹。
2. 打开 **Search** 目录。
3. 点击右下角的 **“加号”** 按钮。
4. 选择 **“新建文件夹”**。
5. 文件夹名称输入你想搜索的内容，例如片名、演员、类型或年份。
6. 点击 **“创建”**。
7. 打开这个新文件夹。

Zwind 会用这个文件夹名称去 Emby Server 搜索，并把搜索结果显示成可浏览的媒体目录。

## 在电视或其他 WebDAV 客户端中播放

如果你想让电视、盒子或其他播放器通过 WebDAV 访问 Emby，需要让这些设备能访问 Zwind 的 WebDAV 地址。

1. 在 Zwind 首页选中服务器。
2. 确认服务器已经启动。
3. 查看服务器显示的 **“地址”**。
4. 在电视或播放器的 WebDAV 设置中填入这个地址。
5. 打开你创建的 Emby 路径。

需要让局域网设备访问时，可以把服务器绑定到 `0.0.0.0` 或当前局域网地址。

## 媒体传输方式怎么选

在 Emby 解析器绑定里，**“媒体传输方式”** 会影响第三方客户端打开视频时的行为。

| 方式 | 适合场景 |
| --- | --- |
| **代理媒体** | 推荐优先使用。客户端只需要访问 Zwind，Zwind 再去 Emby 读取媒体流。 |
| **重定向到 Emby** | 客户端本身可以直接访问 Emby Server，并且能处理 Emby 返回的播放地址。 |
| **自动** | Zwind 根据媒体源情况自动选择代理或重定向。 |

## 刷新 Emby 缓存

Zwind 会缓存一部分 Emby 目录、元数据、图片和播放信息。

如果你刚刚在 Emby 里新增了媒体，或者修改了媒体库内容，但 Zwind 里还没显示出来，可以这样刷新：

1. 在 Browser 中进入 **Emby** 文件夹，或进入 Emby 下的某个子目录。
2. 点击右下角的 **“加号”** 按钮。
3. 选择 **“刷新 Emby”**。
4. 等待提示 **“Emby 缓存已刷新”**。
5. 回到当前目录下拉刷新，或重新进入目录。

如果仍然没有变化，可以回到首页重启这个 WebDAV 服务器。

---

## 简单总结

整个流程可以理解为：

**创建本地 WebDAV 服务器 → 添加数据文件夹 → 创建 Emby 文件夹 → 绑定 Emby 媒体库解析器 → 浏览和播放媒体**

后续你只需要打开 Zwind 的 Emby 文件夹，就可以像浏览 WebDAV 目录一样使用 Emby 媒体库。
